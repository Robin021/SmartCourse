# 一、产品概述与战略目标

# （一）产品定位

以“教育AI咨询+课程生成”为双引擎的SaaS产品，面向K12学校及教育管理部门，提供从课程顶层设计、教学内容生成到学情评估的全流程智能化解决方案，实现教育理念与实践的数字化落地。

# （二）愿景与使命

1. 愿景：成为教育智能化转型的核心驱动力，让每所学校都能拥有个性化优质课程体系  
2. 使命：通过AI技术链接教育哲学、校情特色与教学实践，推动从课程数字化到教育智能化的跨越

# （三）产品核心价值与差异化

1. 双引擎协同：AI咨询模块提供专业诊断（如SWOT量化分析），课程生成模块落地实践方案，形成“诊断-设计-实施-评估”闭环  
2. 在地化适配：深度融合地域文化（如运河文化）、校情特色（如红色基因）与政策要求，生成专属课程体系  
3. 全流程共创：教师可通过对话式交互参与课程设计全环节，AI动态响应修改需求，平衡智能化与个性化  
4. 数据驱动优化：通过课程评估雷达图、学情分析等可视化工具，实现课程持续迭代

# （二）典型使用场景

# 1. 课程顶层设计场景

1. 校长输入学校SWOT数据、教育哲学倾向、地域文化等信息  
2. AI生成《学校教育哲学陈述》《办学理念阐释》《课程结构方案》等系列文档  
3. 教研主任补充学段细节，教师团队在线协作修改，形成最终方案

# 1. AI诊断报告场景

1. 学校上传现有课程体系与学情数据  
2. AI通过RAG检索政策标准与标杆案例，诊断课程目标与育人目标的差距点  
3. 输出含 SWOT 量化分析、薄弱维度预警的诊断报告，附优化建议

# 1. 区域对比场景

1. 教育局导入区域内多所学校课程数据  
2. AI 聚合分析各校课程特色、五育覆盖度、目标达成率等指标  
3. 生成区域课程质量雷达图，识别优势领域与改进方向

# （三）核心功能说明

# 1. AI共创工作区

1. 对话式交互：教师以自然语言提出修改需求（如“增加运河文化实践模块”），AI实时迭代内容  
2. 版本管理：自动保存每轮修改记录，支持回溯对比（如“恢复Q3.2核心概念初稿”）  
3. 实时协作：多人在线标注文档，显示修改者身份与意见，支持投票决策

# 2. 课程设计生成和导出

1. 生成格式：支持Word（含标准教育文档排版）、PDF、PPT（可视化结构图）  
2. 导出选项：全流程打包导出（十阶段成果）、单阶段导出、自定义组合导出  
3. 模板适配：内置教育局备案模板、校本科研模板等3类标准模板

# 3. 课堂教学内容生成和导出

1. 生成内容：基于课程目标的教案、课件、探究任务单、作业设计  
2. 学段适配：自动匹配低/中/高学段认知水平（如低年级增加图文比例）  
3. 导出功能：支持分课时导出、按学科分类导出，课件兼容PPT/Keynote格式

# 4. 教师辅助工具集

1. 课程规划工具：输入主题自动生成跨学科教学序列（如“运河文化”  $\rightarrow$  语文  $+$  历史  $+$  科学融合方案）  
2. 学情分析工具：上传作业/测试数据，生成学生能力雷达图与个性化辅导建议  
3. 资源匹配工具：根据课程模块推荐优质素材（如视频、实验手册、外部场馆资源）  
4. 作业生成工具：按知识点生成分层作业，支持自动批改客观题

# 5. 学生成长平台

1. 成长档案：记录课程参与数据、评价结果、获奖情况，生成电子成长手册  
2. 目标追踪：可视化展示学生与育人目标的差距（如“数码创造力”达标进度）  
3. 互动功能：学生上传实践成果（如项目报告、手工作品），教师在线点评

# 6. 课程评估与雷达分析仪表板

1. 核心指标：五育覆盖度、目标达成率、学生参与度、教师实施难度  
2. 可视化呈现：雷达图（校际对比）、趋势图（月度/学期变化）、热力图（模块优势分布）  
3. 预警功能：指标低于阈值时自动提示（如“德育模块达成率低于  $60\%$ ”）

# 四、关键前端页面交互逻辑设计

# （一）登录页

1. 进入页面：显示产品Logo与“教育AI共创平台”标语，支持账号密码登录与验证码登录  
2. 角色选择：登录后弹出角色选择框（校长/教研主任/教师/教育局），不同角色跳转至对应工作台  
3. 权限验证：系统校验角色权限，如教师端隐藏区域对比功能  
4. 记住登录：提供“记住账号”选项（默认7天），支持“忘记密码”（通过绑定手机号重置）

# （二）十阶段课程设计输入和内容生成页

1. 阶段导航：顶部显示十阶段进度条（如“Q1学校课程情境[已完成]  $\rightarrow$  Q2教育哲学[进行中]”），支持直接跳转  
2. 输入区域：

1. 选择题：弹出下拉列表（如Q2哲学理论选择），可多选，实时显示已选项  
2. 开放式输入：提供文本框（支持换行/粘贴），Q1SWOT表格采用可编辑表格组件（内外维度分栏）  
3. 补充说明：每个输入项旁有“示例”按钮，点击显示参考案例（如Q3.1价值观问题示例)

3. 生成交互：点击“生成内容”按钮后显示加载动画，生成完成后展示结果面板（左侧输入/右侧输出）  
4. 修改交互：输出面板提供“修改建议”文本框，输入需求后点击“迭代生成”，新增内容标红显示

# （三）AI共创工作区

1. 布局设计：左侧对话区（显示历史交互记录），右侧内容编辑区（当前文档）  
2. 对话交互：顶部搜索框输入需求（如“优化Q6.3特色目标”），AI回复显示在对话区，支持追问（如“增加劳育相关描述”）  
3. 编辑功能：内容编辑区支持富文本操作（加粗、插入表格/图片），点击“应用AI建议”自动更新文档  
4. 协作提示：多人协作时，他人修改内容以蓝色高亮显示，弹出“XXX正在编辑”提示

# （四）课程设计生成和导出页

1. 成果预览：左侧显示十阶段成果列表，点击某阶段右侧显示预览内容（支持翻页）  
2. 导出设置：顶部选择导出格式（Word/PDF/PPT）、导出范围（全选/自定义勾选）、模板类型  
3. 操作反馈：点击“开始导出”后显示进度条，完成后弹出“导出成功”提示，支持“立即下载”/“发送至邮箱”

# （五）课堂教学内容生成和导出页

1. 课程关联：自动显示当前关联的课程目标（如“Q7.2 智慧维度高年级目标”），支持切换关联目标  
2. 内容选择：勾选需生成的内容类型（教案/课件/作业），点击“生成”后显示内容列表  
3. 个性化设置：提供“难度调整”滑块（低/中/高）、“素材类型”选项（图文/视频/实验）  
4. 导出交互：同课程设计导出，支持按课时批量导出

# （六）教师辅助工具页

1. 工具导航：左侧分类显示工具列表（课程规划/学情分析/资源匹配/作业生成），点击切换  
2. 操作流程：

1. 选择工具  $\rightarrow$  输入参数（如“作业生成”需选择知识点/难度）  $\rightarrow$  点击生成  $\rightarrow$  预览修改  $\rightarrow$  导出

1. 结果展示：学情分析工具生成雷达图，支持点击指标查看详细说明（如“逻辑推理能力薄弱原因”）

# （七）课程评估与雷达分析仪表板

1. 数据筛选：顶部选择时间范围（月度/学期/学年）、对比对象（本校 vs 区域平均/标杆校）  
2. 图表交互：

1. 雷达图：鼠标悬停显示具体数值（如“美育覆盖度  $85\%$ ”）  
2. 趋势图：支持缩放查看细节，点击节点显示对应时期改进建议

# 五、前端交互与用户体验（UX/UI）

# （一）设计原则

1. 教育适配性：采用简洁专业的视觉风格，避免过度花哨设计  
2. 高效性：核心操作（如生成/导出）不超过3步，关键按钮突出显示  
3. 容错性：输入错误时提供明确提示（如“SWOT外部维度需填10项”），支持撤销操作

# （二）页面具体设计

# 1. 整体视觉规范

1. 主色调：教育蓝 (#1E88E5)、活力绿 (#4CAF50)，辅助色为浅灰 (#F5F7FA)  
2. 字体：标题用微软雅黑（16-20pt），正文用宋体（12-14pt），符合教育文档阅读习惯  
3. 图标：采用线性图标（如课程用、评估用）, 统一尺寸  $(24 \times 24 \mathrm{px})$

# 4. 登录页设计

1. 布局：左侧产品介绍（配教育场景图片），右侧登录表单（居中对齐）  
2. 表单元素：账号/密码输入框（带清除按钮）、验证码输入框（含刷新按钮）、角色选择下拉框（带图标）  
3. 交互细节：输入错误时边框变红，显示提示文字（如“密码长度不足8位”）

# 5. 十阶段课程设计页设计

1. 顶部：进度条（已完成阶段为蓝色，进行中为绿色，未开始为灰色）+帮助按钮（问号图标，点击显示该阶段指南）  
2. 左侧输入区：标题（如“Q1学校课程情境分析”）+输入项（带示例按钮）+保存按钮（自动保存时显示“已保存”标识）  
3. 右侧输出区：标题（如“《学校课程资源分析》”）+内容面板（支持折叠/展开章节）+操作栏（修改/导出/分享按钮）

# 6. AI共创工作区设计

1. 左侧对话区：顶部搜索框（占位符“请输入修改需求，如增加地域文化元素”）+对话列表（AI回复带机器人图标，用户输入带头像）  
2. 右侧编辑区：顶部文档标题（可修改）+富文本工具栏（加粗/斜体/插入表格等）+内容区域（AI生成内容默认左对齐，修改内容标红）  
3. 底部操作栏：应用建议/保存版本/邀请协作按钮，显示当前在线人数

# 7. 课程评估仪表板设计

1. 顶部：筛选栏（时间/对比对象）+导出报告按钮  
2. 中部：核心指标卡片（5个，显示五育达成率，数值异常时卡片边缘变红）  
3. 下部：雷达图（占左侧2/3）+改进建议列表（占右侧1/3，支持点击“查看详情”跳转)

# （二）RAG工作流详解

# 1. 检索阶段

触发条件：用户完成某阶段输入（如Q2.2输入地域文化）

检索范围：私有知识库（校史数据、过往方案）、公共知识库（政策文件、哲学理论库、地域文化数据库）

检索策略：基于关键词匹配（如“运河文化 + 跨学科”）与语义相似度排序（Top5相关文档)

# 2. 增强阶段

信息整合：提取检索结果核心信息（如运河文化历史脉络、相关教学案例）

Prompt 增强：将用户输入 + 整合信息 + 角色指令组合为增强 Prompt（如“作为教育哲学专家，结合江南运河文化与实用主义关键词，生成在地化主张”）

# 3. 生成阶段

模型调用：将增强 Prompt 发送至阿里千问 3，设置参数（温度 0.7，最大长度 2000 字）

内容适配：按教育文档规范格式化输出（如标题层级、段落间距）

# 4. 验证阶段

合规校验：检查内容是否符合教育方针（如“五育并举”提及情况）

适配性校验：比对内容与校情输入（如是否呼应“年轻教师”优势）

异常处理：校验不通过则返回“需补充XX信息”提示，重新触发检索-生成流程

# 5. 回写阶段

结果存储：将生成内容存入 MongoDB，关联用户 ID 与阶段标识

知识库更新：将优质生成内容（经人工审核后）补充至私有知识库，优化后续检索效果

# （四）模型管理与安全隔离方案

# 1. 混合模型架构

1. 云端模型：阿里千问3提供基础生成能力，部署于阿里云专有网络（VPC）  
2. 私有模型：针对敏感校情数据（如学生画像），部署轻量微调模型（基于千问3蒸馏)  
3. 调用逻辑：一般内容（如哲学关键词提炼）调用云端模型；敏感内容（如学情诊断）调用私有模型

# 2. 安全隔离措施

1. 网络隔离：私有模型部署于独立子网，仅通过内网API与业务服务通信，禁止公网访问  
2. 数据隔离：用户数据按学校ID分区存储，模型调用时仅传入本学校数据，避免跨校泄露  
3. 权限控制：模型操作需通过二次认证（如教研主任授权），记录操作日志

# 七、后端逻辑与数据流设计

# （一）数据流说明

# 1. 用户输入  $\rightarrow$  AI 处理流

1.前端验证：检查输入完整性（如Q1SWOT是否填全20项），不完整则返回提示  
2. 数据传输：通过 HTTPS 加密传输至 API 网关，附加用户 Token 与角色标识  
3. 服务路由：API网关将请求转发至对应业务服务（如Q2请求→课程设计服务）  
4.AI调用：业务服务构造Prompt，调用AI中台接口，传入RAG检索参数  
5. 结果处理：AI中台返回生成内容，业务服务进行格式校验与合规检查

# 2. 内容存储  $\rightarrow$  可视化流

1. 存储：合规内容按“用户ID-阶段ID-版本号”路径存入MongoDB，附件（如课件）存入OSS  
2. 数据加工：评估服务定期读取存储数据，计算核心指标（如五育覆盖度）  
3. 可视化转换：加工后的数据传入前端，通过ECharts生成雷达图/趋势图  
4. 展示：前端按角色权限展示可视化结果（如教育局显示区域数据，教师显示班级数据）

# （二）隐私合规与内容审查机制

# 1. 隐私合规

1. 用户授权：首次使用时弹窗展示《隐私政策》，明确告知数据用途，获得同意后方可收集数据  
2. 数据留存：学生个人数据留存至毕业 + 2 年，到期自动匿名化处理  
3. 合规依据：符合《个人信息保护法》《教育数据安全管理办法》等法规

# 2. 内容审查机制

1. 事前审查：AI生成内容触发关键词过滤（如敏感政治词汇），违规内容自动拦截  
2. 事中监控：多人协作时，敏感内容修改需管理员审批  
3. 事后审计：每周生成内容合规报告，重点核查教育方针符合性

# （三）日志审计与访问控制方案

# 1. 日志审计

1. 日志内容：记录用户操作（登录/生成/导出）、AI调用参数、数据修改轨迹，包含操作人、时间、IP地址  
2. 存储要求：日志加密存储于独立服务器，留存至少6个月  
3. 审计功能：支持按用户/操作类型/时间检索日志，异常操作（如批量导出敏感数据）自动告警

# 2. 访问控制

1. 角色权限：基于RBAC模型，校长拥有全权限，教师仅可操作本学科课程  
2. 设备绑定：支持绑定常用设备，新设备登录需短信验证  
3. 会话管理：登录会话超时30分钟自动失效，强制重新登录
